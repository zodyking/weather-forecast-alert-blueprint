Overview

Fetches daily and hourly forecasts, crafts a concise forecast, optional AI rephrase that preserves facts, then speaks on selected speakers with a pre-roll delay.
Precipitation callouts require hourly precipitation_probability. If missing, the precip line is skipped.

Requirements

  • weather entity supporting weather.get_forecasts (daily + hourly).
  • tts engine and at least one media_player.
  • Optional: an ai_task entity for rephrasing; Conversation integration for voice.

If any branch is true, the corresponding action sequence runs.

Time-based / Sensor-triggered
  1. weather.get_forecasts daily → d; hourly → h
  2. Compose message: greeting + day; Right now (cond/temp/humidity); Today (if condition differs, high, low); First precip window within !input hours_ahead with time and % if ≥ !input precip_threshold
  3. Optional AI pass (ai_task.generate_data) if !input use_ai_rewrite and !input ai_task_entity
  4. media_player.volume_set → !input volume_level
  5. For each !input speakers: delay !input preroll_ms ms, then tts.speak with options.voice = !input voice
Current change
  1. On weather entity state change
  2. Compose: “It’s now <temp> degrees and <condition>.”
  3. Optional AI pass
  4. media_player.volume_set → !input current_change_volume_level
  5. Per-speaker delay + tts.speak
Upcoming change
  1. Hourly fetch → next precip that begins within !input minutes_before_announce if not precipitating now
  2. Speak: “<rain/snow> in about <minutes> minutes.”
  3. Optional AI pass
  4. media_player.volume_set → !input upcoming_change_volume_level
  5. Per-speaker delay + tts.speak
Alarm webhook
  1. Daily + hourly fetch; personalize with !input personal_name
  2. Optional AI pass; prefix “Good morning <name> …”
  3. media_player.volume_set → !input alarm_volume_level
  4. Per-speaker delay + tts.speak
Voice satellite
  1. On phrase match in !input conversation_command build standard forecast text
  2. Optional AI pass
  3. Return text with set_conversation_response

Webhook alarm: Android & iOS

Canonical URL format

http://homeassistant.local:8123/api/webhook/<WEBHOOK_ID_FROM_INPUT>

Android (Shortcuts or Automation app)

  1. Create a new shortcut/automation named Weather Alarm.
  2. Add an HTTP POST action.
  3. URL: use the canonical webhook URL above with your webhook_id.
  4. Headers: none required when on local network.
  5. Body: leave empty.
  6. Save and run the shortcut to trigger the blueprint’s “Alarm webhook” path.

If you use Nabu Casa remote URL, replace host with your remote base URL.

iOS (Shortcuts app)

  1. Open Shortcuts → “+” → Add Action → “Get Contents of URL”.
  2. Method: POST.
  3. URL: http://homeassistant.local:8123/api/webhook/<WEBHOOK_ID_FROM_INPUT>
  4. Request Body: None.
  5. Optional: Add “Set Volume” before the request to adjust device volume.
  6. Save as Weather Alarm. Run it to test.

Troubleshooting

  • No audio: Confirm speakers includes at least one media_player. Increase preroll_ms if the first syllable is cut.
  • No precip mention: Your hourly source may lack precipitation_probability. The line is omitted by design.
  • AI errors: Turn off use_ai_rewrite to confirm base message. Check your ai_task entity.
  • Voice phrases not responding: Ensure Conversation integration is enabled and phrases match lines in conversation_command.

FAQ

How do I schedule every 3 hours at minute 3?
Set hour_pattern to /3 and minute_offset to 3.
How do I change the voice?
Set voice to a voice ID your TTS engine supports. Example: en_US-carlin-high.
What’s the correct webhook URL?
Use http://homeassistant.local:8123/api/webhook/<WEBHOOK_ID_FROM_INPUT>. Replace the placeholder with your blueprint input.
